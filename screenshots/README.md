# Screenshots

Visual walkthrough of the lab. Each image is rendered from the data and scripts in this repository using the fictional **QueensTech Office Services** sample data — no real names, emails, or credentials appear.

These are linked from the main [`README.md`](../README.md#screenshots).

## Included

| File | Shows |
|------|-------|
| `ticket-dashboard.png` | The Level 1 queue — 30 tickets with category, P1–P4 priority, status, and escalation, plus SLA metrics. |
| `sample-ticket.png` | One P1 ticket (TKT-015, phishing / BEC) end to end: description, impact, troubleshooting, root cause, resolution, and user response. |
| `m365-user-list.png` | Microsoft 365 / Entra ID admin-center view of the 25-user tenant — accounts, licenses, MFA state, and status. |
| `powershell-output.png` | `Get-DeviceHealth.ps1` and `Check-NetworkStatus.ps1` running with clean, color-coded output. |
| `kb-article.png` | A rendered knowledge base article (KB-001, Password Reset) — problem, symptoms, fix, and escalation notes. |

## Notes

- All screens use the fictional QueensTech sample data (`sample-data/`, `tickets/`, `knowledge-base/`). Keep any additional screenshots free of real names, emails, or credentials.
- The dashboard, ticket, M365 user list, and KB views are presentation mock-ups of the underlying Markdown/CSV records; the PowerShell screenshot reflects the actual script output.
