# Workflow — Access Request

**Owner:** IT Help Desk (Level 1) | **Trigger:** User requests access to a folder, app, group, or shared resource.

> Goal: grant the right access to the right person — with approval and least privilege.

---

## Step 1 — Capture the Request
- [ ] What resource? (shared folder, app/CRM, shared mailbox, group, drive)
- [ ] Business reason and role justification.
- [ ] Requested by the user or their manager?

## Step 2 — Approval
- [ ] Access to **secured/sensitive** resources requires **manager (or data owner) approval**.
- [ ] Obtain approval in writing (email/ticket) and **archive it** with the ticket.
- [ ] Deny or escalate if approval is missing or the request seems excessive.

## Step 3 — Provision (Least Privilege)
- [ ] Grant access via **security group membership**, not one-off permissions, wherever possible.
- [ ] Add the user to the correct group (e.g., Finance Team, Sales Team, Shared Mailbox Users).
- [ ] Avoid over-granting — give only what the role needs.

## Step 4 — Verify
- [ ] Have the user **sign out and back in** for the permission to apply.
- [ ] Confirm the user can open/use the resource.
- [ ] Confirm they can **not** see anything beyond what was approved.

## Step 5 — Record
- [ ] Log what was granted, by whose approval, and when.
- [ ] Close the ticket.

## Escalation
- Elevated/admin permissions or tenant-level app assignments → **Level 2**.
- Paid app licenses required → **Level 2 / budget**.
