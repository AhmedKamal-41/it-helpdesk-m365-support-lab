## User Description
"I've taken on accounts payable duties and need access to the Finance > AP shared folder, which I can't open right now."

## Business Impact
User cannot reach files needed for their new responsibilities. Blocked for part of their role.

## Troubleshooting Steps
1. Confirmed the request via the Access Request Workflow — access to secured data requires manager approval.
2. Obtained written approval from Priya Nair (Finance Director) via email.
3. Verified the user was not already in the "Finance Team" security group.
4. Added the user to the "Finance Team" security group that grants the folder permission.
5. Had the user sign out and back in, then confirmed they could open the AP folder.

## Root Cause
User's new duties required folder access that had not yet been provisioned. Standard access-provisioning request.

## Resolution
With manager approval on file, added the user to the appropriate security group. User confirmed they can now open and edit files in the Finance AP shared folder.

## User-Facing Final Response
> Hi Robert, your access to the Finance AP shared folder is now active (approved by Priya). Please sign out and back in once so the permission fully applies, then you're good to go. — IT Help Desk

## Prevention / Follow-Up Notes
- Approval email archived with the ticket for audit purposes.
- Access granted via group membership, not direct folder permissions, per best practice.
