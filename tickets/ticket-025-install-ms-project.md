## User Description
"I need Microsoft Project to build out our operations schedules. It's not on my machine."

## Business Impact
Low urgency but needed for a project-planning initiative. Not blocking daily work.

## Troubleshooting Steps
1. Identified Microsoft Project as a **licensed add-on**, not part of the standard M365 plan.
2. Confirmed no spare Project license was currently assigned in the M365 admin center.
3. **Escalated to Level 2** — purchasing/assigning a paid license requires IT Manager and budget approval.
4. IT Manager approved and assigned a Project Plan license to the user.
5. Installed Microsoft Project from the M365 portal and verified activation.

## Root Cause
Requested software required a separate paid license that had to be procured and assigned — outside Level 1 authority.

## Resolution
After Level 2 approval and license assignment, installed Microsoft Project and confirmed it activated under the user's account. User opened a test project file successfully.

## User-Facing Final Response
> Hi Daniel, Microsoft Project needed a separate license, which our IT Manager approved. It's now installed and activated on your machine and ready for your operations schedules. Let me know if you'd like a quick orientation to the app. — IT Help Desk
